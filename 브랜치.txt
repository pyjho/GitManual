

1. git -c init.defaultBranch=main init
 명령어를 풀어보면 이렇게 됩니다:
 git init
 새로운 Git 저장소를 초기화하는 기본 명령어예요. 실행하면 .git 디렉토리가 생성되고, 해당 디렉토리가 Git 저장소로 인식됩니다.

 -c init.defaultBranch=main
 git 실행 시 임시로 특정 설정(config)을 적용하는 옵션입니다. 
 여기서는 init.defaultBranch=main 설정을 적용한 것이고, 이는 초기화될 때 기본 브랜치 이름을 main으로 하라는 의미입니다.

Git 2.28 버전 이후부터는 기본 브랜치 이름이 master 대신 main으로 바뀌는 추세인데, 아직 환경에 따라 master로 만들어지는 경우가 있습니다.
이때 -c init.defaultBranch=main을 붙이면, 전역 설정을 바꾸지 않고도 딱 이번 init 실행에 한해 main 브랜치를 기본 브랜치로 생성하게 됩니다.

2. git branch -M main
현재 체크아웃된 브랜치명을 강제로 main으로 변경.
